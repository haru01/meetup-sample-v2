# ドメインスターター — よくあるドメインの候補イベントリスト

フレーミングでドメインが確定したら、近いスターターを選んで候補リストの下書きに使う。
ユーザーへの提示前に、スコープ・アクター・フローに合わせて不要なイベントを削除・調整すること。

---

## 転職サービス（求職者フロー：登録〜内定）

```
─── 登録フェーズ ───
[1] 会員登録が完了した
[2] プロフィールが入力された
[3] 職歴・スキルが登録された
[4] 希望条件が設定された

─── AI初期コンタクトフェーズ ───
[5] AIキャリアアドバイザーが初期コンタクトを開始した
[6] チャットで通知設定が完了した

─── AIマッチング（定期）フェーズ ───
[7] AIマッチングバッチが実行された
[8] おすすめ求人リストが生成された
[9] おすすめ求人がチャットで通知された

─── 応募フェーズ ───
[10] 求人に応募した
[11] 書類選考結果が通知された

─── 面接フェーズ ───
[12] 面接日程が調整された
[13] 面接が実施された
[14] 選考結果が通知された

─── 内定フェーズ ───
[15] 内定を受諾した
[16] 内定を辞退した
```

**よくある追加イベント（スコープ次第）：**
- スカウトが届いた（企業起点フロー）
- 応募が取り消された
- 複数回面接が実施された
- 入社日が確定した

```
─── 商品探索フェーズ ───
[1] 商品が検索された
[2] 商品ページが閲覧された
[3] 商品がカートに追加された

─── 購入フェーズ ───
[4] 注文が確定した

[6] 注文確認メールが送信された

─── 配送フェーズ ───
[7] 倉庫で出荷準備が完了した
[8] 商品が発送された
[9] 配送追跡番号が通知された
[10] 商品が配達された

─── アフターフェーズ ───
[11] 購入レビューが投稿された
[12] 返品リクエストが申請された

```

**よくある追加イベント：**
- クーポンが適用された
- 在庫切れが通知された
- お気に入りに追加された
- 定期購入が設定された

---

## 使い方のポイント

- スターターはあくまで「下書き」。そのまま使わず、ドメインに合わせてカスタマイズする
- フレーミングで確認した「スコープ」「アクター」「ゴール」に合わないイベントは削除する
- AIエージェント・バッチ処理が登場する場合は、対応するフェーズを追加する
- イベント数が多すぎる場合は、最も重要なコアフロー（ハッピーパス）に絞る
